# CapCut Deliverables Skill ⭐

**인스타 발행용 메타데이터 파일 생성 — 제목 / 캡션 / 스크립트**.

CapCut 드래프트에는 `--title` 플래그로 타이틀 텍스트만 들어가고, 실제 발행·반응 측정에 필요한 **제목 + 캡션 + 스크립트 + 최종 영상**은 별도 `deliverables/` 폴더에 파일로 기록해야 합니다. 이 스킬은 (a) `title.txt` 후킹 한 줄, (b) `ig_caption.txt` 400~600자 본문 + 해시태그 5개, (c) `script.txt` SRT 평문 변환을 담당합니다.

관련 스킬: [`capcut-pipeline`](../capcut-pipeline/SKILL.md) (드래프트 생성), [`capcut-fx`](../capcut-fx/SKILL.md) (FX 주입), [`capcut-subtitle`](../capcut-subtitle/SKILL.md) (자막 타이밍).

---

## 📁 출력 파일 위치

```
output/<name>/deliverables/
├── title.txt        # 영상 제목 (한 줄, 후킹)
├── ig_caption.txt   # 인스타 캡션 (400~600자 + 해시태그 5)
├── script.txt       # SRT 평문 (선택, script.txt 생성 섹션에서 변환)
└── final.mp4        # CapCut 내보내기
```

**언제 만드나**: STT + transcript 교정 완료 후, scene_designer plan과 병행. CapCut 내보내기 전에 `title.txt` / `ig_caption.txt` 2개는 미리 작성.

**누가**: Claude Code 메인이 `transcript_wrapped.srt`를 읽고 직접 작성 (별도 API 키 불필요).

## ⛔ `title.txt` 규칙

- **한 줄** — 말미 개행 없음 (빈 줄 금지)
- **10단어 / 20자 이하** — 첫 3초 스크롤 방어
- **후킹 요소 1개 이상 포함**:
  - 의문형 (`?`)
  - 구체 숫자 (`3가지`, `10배`, `500만 건`)
  - 반전/대조 (`~인데 사실은`, `~인 줄 알았죠`)
  - "비밀/공식/이유" 키워드
- **핵심 키워드 포함** (검색·추천 알고리즘 대응)
- 영상 첫 씬 narration과 메시지 일치 — **클릭베이트 금지**

✅ 좋은 예:
```
답장 빨라지는 이메일 제목 3가지
일잘하는 사람은 뇌를 이렇게 씁니다
```

❌ 나쁜 예:
```
이메일에 대해 이야기해 봅시다        ← 후킹 없음
충격! 이메일 하나로 인생 역전!!!    ← 금지 표현 + 클릭베이트
```

## ⚡ `ig_caption.txt` 규칙 (가장 중요)

**본문 길이**: 600~1300자 (해시태그 제외)
**캡션은 그 자체로 "저장하고 싶은 가이드"여야 한다** — 영상을 안 본 사람도 이 글만 읽고 핵심을 이해하고 바로 실천할 수 있을 만큼 **체계적이고 상세**해야 함. 대본을 그대로 옮긴 요약은 실패다 (→ §0.5 필독).

### 0. ⚡⚡ 첫 줄 = 후킹 (가장 중요 — 위반 시 전면 재작성)

⛔ **인스타/페북 피드는 캡션의 첫 1줄(약 30~40자)만 노출하고 나머지는 `… 더 보기`로 접는다.** 즉 **첫 줄이 캡션 전체의 클릭률을 결정**한다. 첫 줄이 약하면 본문이 아무리 좋아도 아무도 펼쳐 보지 않는다.

> 사용자 지침: **"캡션은 첫 줄이 후킹이 잘 되어야 해. 캡션을 눌러서(더 보기) 보고 싶은 마음이 들게."**

**첫 줄 작성 규칙**:
1. **궁금증 갭(curiosity gap)을 만든다** — "진짜 이유가 따로 있다 / 사실은 ~가 아니다"처럼 답을 살짝 감춰 "그게 뭔데?"를 유발. → 더 보기를 누르게 하는 1순위 장치.
2. **통념 반전 또는 강한 공감 페인포인트**로 시작 — 타깃이 "내 얘기다" 하고 멈추게.
3. **한 줄 안에서 완결**(30~40자 이내, 줄바꿈 전에 끝) — 잘려서 의미가 깨지면 안 됨. 첫 문장 뒤 **빈 줄**로 분리.
4. **영상 첫 3초(훅)·제목과 다른 변주** — 복붙 금지. 같은 메시지를 다른 각도로.
5. **답/결론을 첫 줄에서 다 말하지 말 것** — 본문을 펼쳐야 알 수 있게 여지를 남긴다.
6. 톤·금지표현 규칙(아래 1·2번)은 첫 줄에도 그대로 적용 (반말·`충격!`·이모지 금지).

| ✅ 후킹 강한 첫 줄 | ❌ 약한 첫 줄 |
|---|---|
| 열심히 해도 아무도 안 알아주는 진짜 이유, 따로 있습니다. | 오늘은 번아웃에 대해 이야기해 볼게요. |
| 번아웃이 온 진짜 원인, 사실 일이 많아서가 아니거든요. | 직장 생활 힘드시죠? 다들 그래요. |
| 인정받지 못해 서운하셨다면, 그건 절대 유난이 아니에요. | 제가 요즘 느낀 점을 공유합니다. |

→ 왼쪽은 "그래서 이유가 뭔데?"로 **더 보기**를 누르게 만들고, 오른쪽은 정보가 0이라 스크롤로 지나친다.

**첫 줄 = 미끼, 본문 = 미끼를 문 보상.** 첫 줄로 클릭을 유도했으면 본문은 그 기대(§0.5의 체계적 가이드)로 반드시 보답해야 한다. 첫 줄만 강하고 본문이 대본 요약이면 "낚였다"는 인상만 남는다.

### 0.5. ⛔⛔ 캡션 ≠ 대본 요약 (독립 콘텐츠 원칙 — 2026-06-08 사용자 피드백)

> 사용자: **"캡션 내용은 영상 내용을 매우 상세하고 정리되고 체계적으로 보여줘야 해. 예시도 있고, 캡션 내용 자체가 도움이 되는 정보여야 하는데 지금은 그냥 대본 요약한 느낌이야."**

⛔ **나레이션 문장을 그대로 풀어쓰지 말 것.** 영상에서 말로 한 문장을 캡션에 똑같이 옮기면 "대본 요약"이 된다 — 영상 본 사람에겐 중복, 안 본 사람에겐 맥락 없는 파편.

✅ **캡션은 영상 주제를 "글로 다시 설계한 독립 가이드"다.** 영상이 30~80초 안에 비유로 던진 인사이트를, 캡션은 차분하게 **번호·단계·구분·예시**로 풀어 "저장하고 싶은 글"로 만든다. 영상을 안 본 사람도 이 글만으로 핵심을 이해하고 실천할 수 있어야 한다.

| ❌ 대본 요약 (하지 말 것) | ✅ 독립 가이드 (지향) |
|---|---|
| "강점은 볼륨이랑 비슷해요. 적당히 켜면 음악, 끝까지 올리면 소음이 되거든요." (영상 문장 그대로) | "강점에는 '적정 볼륨'이 있습니다. **볼륨 1~3**은 강점이 무기가 되는 구간, **볼륨 9~12**는 같은 강점이 나와 주변을 갉아먹는 구간이에요. 예를 들어 책임감이라면 — 볼륨 3은 '내 일은 끝까지 책임진다', 볼륨 10은 '남의 일까지 떠안아 번아웃'입니다." |
| 영상이 한 말을 나열 | 개념 정의 → 분류 → 구체 예시 → 적용 |

**가이드형 캡션이 반드시 담아야 할 5요소** (영상보다 한 발 더 깊게):
1. **개념을 한 줄로 정의** — 영상이 비유로 던진 걸 캡션은 명확한 정의로.
2. **체계적 분류·단계** — `①②③` 번호, `A일 때 / B일 때`, `1단계~3단계` 등으로 **구조화**. 줄글 벽글자 금지.
3. **구체적 예시 최소 1개** — 추상 개념을 독자 상황으로 번역 (실제 직군·상황·숫자·대사).
4. **바로 해볼 수 있는 실천법/자가진단** — "오늘 이것부터 해보세요" 수준의 액션 또는 체크 질문.
5. **영상에 없던 한 끗** — 통계·심리학 근거·추가 팁 등 캡션에서만 주는 보너스 (저장률↑).

> 판별 테스트: **캡션에서 영상 나레이션과 90% 겹치는 문장이 보이면 재작성.** 같은 주제를 "다른 문장·다른 구조(번호/예시)"로 풀었는지 확인.

### 1. 톤 & 어체 ⚠️ (위반 시 전면 재작성)

- **전문가가 진심을 담아 이야기하는 어조** — 차분, 신뢰, 진정성
- **기본 격식체** `~합니다 / ~입니다` + **공감·전환 혼용** `~거든요 / ~잖아요`
- **반말/평서형 절대 금지** — `~다 / ~있다 / ~않는다 / ~이다`
- **호칭**: `여러분` 고정 (절대 `당신` 금지. 변주 필요 시 `~하시는 분들`)
- **권장 전환구**: `왜냐하면~`, `그래서~`, `바로~`, `이게 바로~`
- **공감 표현**: `~하시는 분들 같아요`, `~하신 적 있으시죠?`, `~겪어보셨죠?`
- **진정성**: `제 경험으로는~`, `도움이 되셨으면 좋겠어요`

### 2. ⛔ 금지 표현

`충격!` · `대박!` · `무조건` · `100%` · `반드시` · `인생이 바뀝니다` · 이모지 (불릿 `—` `·` 만 허용)

| ✅ 좋은 예 | ❌ 나쁜 예 |
|---|---|
| 결과가 10배 차이나는 이유가 있거든요. | 결과가 10배 차이납니다. |
| 겪어보신 분들 많으시죠? | 당신도 겪어봤을 겁니다. |
| 답장받는 분과 안 받는 분의 차이예요. | 답장 못 받으면 인생이 바뀝니다. |
| 세 가지만 기억해 주세요. | 무조건 이거 3개 외워! |

### 3. 구조 템플릿 (가이드형 — 체계적·상세, §0.5 5요소 반영)

```
[후킹 첫 줄 ⚡ — 궁금증 갭/통념 반전/강한 공감. "더 보기"를 누르게 만드는 30~40자. 답은 감추고, 영상 훅·제목과 다른 변주(§0 필수)]

[개념 정의 — 영상이 비유로 던진 핵심을 명확한 1~2문장 정의로. "사실 ~란 ~입니다"]

[체계적 본문 ⭐ — 캡션의 몸통(가장 길고 가치 있는 부분). 번호·단계·구분으로 구조화 + 각 항목에 구체 예시:
 ① 첫 번째 포인트 — (예시: 실제 상황/직군/숫자)
 ② 두 번째 포인트 — (예시: ~)
 ③ 세 번째 포인트 — (예시: ~)
 또는 'A일 때 / B일 때', '1단계~3단계' 형태]

[자가진단 또는 실천법 — 독자가 오늘 바로 해볼 수 있는 체크 질문/액션]

[영상에 없던 보너스 한 줄 — 통계/심리 근거/추가 팁 (선택, 저장 가치)]

[CTA — 댓글에 "[키워드]" 남겨주시면 [구체 혜택] 보내드릴게요]

#해시태그1 #해시태그2 #해시태그3 #해시태그4 #해시태그5
```

- **체계적 본문(③ 블록)이 캡션의 핵심** — 여기에 번호·예시가 없으면 "대본 요약"으로 전락. 가장 공들일 부분.
- 각 블록·항목마다 **빈 줄** (벽글자 금지, "더 보기" 펼쳤을 때 스캔 가독성)
- 번호 항목은 `①②③` 또는 `1.` `2.` `3.`, 하위 예시는 `—`로 — 모바일에서 줄 단위로 읽히게.
- 마지막 줄은 **CTA** (댓글/저장/팔로우/DM 유도)

### 4. 해시태그 5개 + SEO 키워드 5~6개

- **해시태그**: 본문 아래 공백 줄 후 한 줄로. 한글 우선. 업종 + 주제 + 타깃 조합.
  예: `#일잘러 #이메일잘쓰기 #업무효율 #직장인꿀팁 #생산성`
- **SEO 키워드**: 본문에 자연스럽게 녹이거나, 별도 메타로 보관 (검색·관련 콘텐츠 매칭).

### 5. AI 레이블 (선택)

AI 생성 아바타·TTS 사용 영상이면 본문 말미, 해시태그 위에:
```
※ 본 영상은 AI 기술로 제작되었습니다. (가상인물 포함)
```
실제 인물 촬영본(PROMPTER 류)은 생략.

## 📝 예시 (가이드형 — 권장 품질, §0.5 5요소 반영)

`title.txt`:
```
강점이 약점이 되는 순간
```

`ig_caption.txt`:
```
열심히 잘하는데도 자꾸 지친다면, 문제는 노력의 양이 아닐 수 있거든요.

강점에는 '적정 볼륨'이 있습니다. 같은 강점이라도 어느 칸까지 올려 쓰느냐에 따라 무기가 되기도, 나를 갉아먹는 약점이 되기도 하거든요.

볼륨을 1에서 12까지라고 해볼게요.

1. 볼륨 1~3 (강점이 무기가 되는 구간)
— 책임감이라면 "내 일은 끝까지 책임진다", 꼼꼼함이라면 "중요한 것만 두 번 확인한다" 정도예요. 주변이 나를 믿고 맡길 수 있는 선이죠.

2. 볼륨 9~12 (같은 강점이 약점이 되는 구간)
— 책임감이 과해지면 "남의 일까지 떠안아 번아웃", 꼼꼼함이 과해지면 "사소한 것까지 붙들어 마감을 놓침"이 됩니다. 강점이 새로 약점으로 바뀐 게 아니라, 볼륨만 너무 올라간 상태예요.

내 볼륨이 넘쳤는지 확인하는 법은 의외로 간단해요. '요즘 나를 제일 지치게 하는 일'과 '사람들이 나를 제일 많이 칭찬하는 일'이 같다면, 그 강점의 볼륨이 넘쳤다는 신호거든요.

그럴 땐 강점을 끄는 게 아니라 딱 한 칸만 낮추면 됩니다. 능력은 그대로 두고, "이걸 어디까지 책임질지" 경계만 다시 그어주는 거예요. 오늘은 가장 지치는 일 하나를 골라, 내가 안 해도 되는 부분이 어디인지 한 줄로 적어보세요.

내 강점의 볼륨이 지금 몇 칸인지 궁금하시다면, 댓글에 '볼륨'이라고 남겨주세요. 강점별 적정 볼륨 점검 체크리스트를 보내드릴게요.

#강점 #번아웃 #자기계발 #직장인성장 #감정관리
```

> 위 예시가 좋은 이유: 영상은 "볼륨 비유"를 말로 던지지만, 캡션은 **① 개념 정의("적정 볼륨이 있다") ② 1~3 / 9~12 구간 분류 ③ 책임감·꼼꼼함 구체 예시 ④ 자가진단 ⑤ 오늘 할 액션**으로 재설계했다. 나레이션 문장을 그대로 베끼지 않고 같은 주제를 "읽는 가이드"로 다시 썼다.

## 📝 script.txt 생성 (선택)

`transcript_wrapped.srt`의 타임스탬프 제거하고 본문만 합쳐 `script.txt` 평문을 만듭니다. CapCut에서 NG 씬을 삭제했어도 SRT는 원본이므로 NG 테이크 반복 문자열 남아있을 수 있음 — 필요 시 수동 편집.

### ⛔ Windows cp949 경고 — `PYTHONIOENCODING=utf-8` 필수

Windows 기본 콘솔은 **cp949**라, Python이 한글이 섞인 문자열을 `print` 할 때 em-dash(`—`) · middot(`·`) · 화살표(`→`) 등에서 `UnicodeEncodeError: 'cp949' codec can't encode character ...` 로 종료될 수 있음. 파일 쓰기는 이미 완료된 상태지만 stdout 직렬화 단계에서 터지면서 exit 1이 반환되어 마치 실패처럼 보임. 모든 명령에 프리픽스 권장:

```bash
export PYTHONIOENCODING=utf-8          # 세션 전체
# 또는
PYTHONIOENCODING=utf-8 python ...      # 명령당
```

### SRT → 평문 변환

```bash
PYTHONIOENCODING=utf-8 python -c "
import pathlib
srt = pathlib.Path('output/<name>/subs/transcript_wrapped.srt').read_text(encoding='utf-8')
lines = []
for block in srt.strip().split('\n\n'):
    parts = block.split('\n')
    if len(parts) >= 3:
        lines.append(' '.join(parts[2:]).strip())
plain = ' '.join(lines)
pathlib.Path('output/<name>/deliverables/script.txt').write_text(plain, encoding='utf-8')
print(f'wrote {len(plain)} chars')
"
```
